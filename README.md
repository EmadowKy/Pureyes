# Pureyes 视频问答系统

一个基于 AI 的视频问答平台，你可以上传视频，然后针对视频内容提问，AI 会分析视频并给出答案。

## 项目结构

```
Pureyes/
├── backend/     # 后端服务（Flask）
└── frontend/    # 前端页面（Vue3 + Vite）
```

## 快速启动

### 1. 启动后端

```bash
cd backend
pip install -r requirements.txt
python run.py
```

后端启动后会显示 `Running on http://127.0.0.1:8000`，表示服务正常运行。

### 2. 启动前端

```bash
cd frontend
npm install
npm run dev
```

前端启动后访问 `http://127.0.0.1:5173`

## 功能说明

### 用户功能

- **注册登录**：创建账号、登录系统
- **个人中心**：查看用户信息、视频使用量和存储空间限制
- **退出登录**：安全退出当前账号

### 视频管理

- **上传视频**：支持 MP4 格式，单个文件最大 500MB
- **视频列表**：查看已上传的所有视频
- **视频播放**：点击视频直接播放
- **重命名**：给视频起个方便识别的名字
- **删除视频**：不需要的视频可以删掉
- **使用限制**：每个账号最多 30 个视频，总共 2GB 存储空间

### 视频问答

- **提问**：选中视频后输入问题，AI 会分析视频内容并回答
- **时间戳跳转**：回答中会标注具体时间点，点击可跳转到视频的对应位置
- **实时进度**：提问后能看到 AI 分析的进度
- **历史记录**：所有问答都会保存，可以随时翻看
- **导出记录**：支持把问答记录导出为 JSON 文件
- **统计数据**：能看到总提问数、成功数、失败数

## 技术栈

- 前端：Vue3、Vite、Element Plus
- 后端：Flask
- AI：集成大语言模型进行视频内容分析

## MVA 多视频理解模型工作原理

MVA（Multi-Video Agent）是一个多视频联合理解模型，核心是用视觉语言模型（Qwen-VL）来分析和理解多个视频的内容。整个过程分为三个阶段。

### 模型基础

模型使用 Qwen-VL 作为视觉语言 backbone，能够同时处理图像和文本输入。分析视频时会把视频切分成帧（frame），然后逐帧理解。

每个视频维护两个核心数据：

- **Frame Bank**：存放该视频中挑出来的关键帧，每帧带一个分数（0-1），代表这帧和问题的相关程度
- **Description**：文字描述，记录模型对视频各时间段内容的理解

***

### 第一阶段：问题分析

在正式处理视频之前，模型会先分析用户的问题，理解问题在问什么、需要关注什么类型的视觉信息。这一步输出的"问题分析"会贯穿后续所有环节，作为 ToolAgent 和 DescAgent 的参考。

***

### 第二阶段：初始化

对每个视频依次做初始化。

**1. 均匀采样帧**

从视频开头到结尾，均匀地抽出 8 帧。这个采样覆盖了视频的全程，作为初始观察。

**2. 生成初始描述和评分**

DescAgent 接收这 8 帧图像，结合问题、其他视频的描述（如果有的话），生成一段文字描述这段视频的内容，同时给出一个 0-1 之间的分数。分数越高代表这段视频和问题的关联度越高。

初始描述格式类似：`0.00-120.00: 这段视频展示了... (Score: 0.65)`

**3. 建立 Frame Bank**

每帧的初始分数设为 0.5，和描述一起存入 Frame Bank。Frame Bank 有容量限制（默认 16 帧），超出时按分数从高到低保留。

初始化完成后，每个视频都有一个全局的初始描述，以及一批带有分数的关键帧。

***

### 第三阶段：主循环迭代

这是模型的核心推理过程，最多循环 10 次。

**1. 计算优先级，选择视频**

每个视频有一个动态计算的"优先级分数"，由当前分数和加速度（进步速度）共同决定：

$$P = e^{k \cdot a} \cdot (1 - s)^t$$

其中 $a$ 是加速度（分数变化率），$s$ 是当前分数，$k$ 和 $t$ 是权重参数。分数低（还有提升空间）且进步快的视频会被优先选中处理。

**帧分数评分（ToolAgent）**

每帧的分数由 ToolAgent 结合问题的上下文，从帧图像中判断该帧与问题的相关程度。输出格式为：

```
Scores: [0.65 0.32 0.78 0.45 ...]
```

每个分数都是 0-1 之间的小数，分数越高代表该帧与问题的关联度越大。如果模型未能给出足够的分数，则默认填充 0.5。

**视频片段评分（DescAgent）**

DescAgent 对当前采样的视频片段给出综合评分，输出格式为：

```
Score: 0.72
Video Terminated: False
Global Terminated: False
```

评分标准：

- **0.00**：该片段完全没有有用信息，需要继续探索
- **0.50**：该片段提供了一部分有用线索
- **1.00**：该片段已经包含回答问题所需的充分证据

即使分数为 1.0，也可能在其他视频还需要探索的情况下继续处理（多视频联合分析）。

**加速度计算**

加速度用于衡量模型的"进步速度"，即每次迭代后分数的变化率：

$$a = \frac{s\_{new} - s\_{old}}{\max(s\_{old}, 0.01)}$$

其中 $s\_{new}$ 是当前片段的新分数，$s\_{old}$ 是上一次迭代的分数。如果旧分数过低（小于 0.01），分母会固定为 0.01 防止除零。加速度越高，说明最新采样的片段质量提升越明显。

**2. ToolAgent 决策：采样位置和采样策略**

选中的视频交给 ToolAgent 分析。Agent 拿到：

- Frame Bank 中的已有帧（带分数和时间戳）
- 当前批次刚采样的帧（需要评分）
- 该视频的现有描述、其他视频的描述

Agent 输出一组帧的分数，以及一个决策（Option 1-6）：

- **Option 1**：在目标区间内细粒度采样（默认行为）
- **Option 2**：扩大搜索范围
- **Option 3**：在当前区间继续但换策略
- **Option 4**：跳到视频其他位置
- **Option 5**：全局均匀采样（探索新区域）
- **Option 6**：终止对该视频的分析

同时输出目标时间区间 `[target_start, target_end]`。

**3. 按决策采样新帧**

根据 Option 和目标区间，从视频中抽出 8 帧。

**4. DescAgent 描述新帧并判断终止**

新帧交给 DescAgent，生成新的文字描述，追加到视频的完整描述后面。同时判断：

- **Video Terminated**：该视频已经充分分析，可以终止
- **Global Terminated**：所有视频整体已经充分，可以提前结束

**5. 更新状态**

- 更新 Frame Bank：新帧加入，按分数排序，超容量的淘汰低分帧
- 更新 Description：追加新的描述段落
- 更新分数和加速度
- 重新计算优先级

**6. 终止条件**

满足以下任一条件时循环结束：

- 所有视频都被标记终止
- 达到最大迭代次数（10 次）
- 某视频得分超过 0.8（认为已经充分理解）
- 某视频加速度持续过低（认为没什么可挖的了）

***

### 第四阶段：生成最终回答

所有视频分析完毕后，用最终保留的 Frame Bank 和所有 Description 生成答案。

**1. 筛选关键帧**

Frame Bank 中分数低于 0.5 的帧会被过滤掉。

**2. 给帧加水印**

为保留的帧添加视频标签和时间戳，方便最终回答中引用具体时间点。

**3. 调用 Qwen-VL 生成回答**

把筛选后的关键帧、累积的所有描述、原始问题一起发给视觉语言模型，生成最终的自然语言回答。回答中会包含 `[video:X, time:XX:XX]` 格式的时间戳标记，用户点击可以跳转到对应视频的对应时间点。

***

### 处理流程图

```
用户提问
    │
    ▼
问题分析 ─────────────────────────────────────────┐
    │                                               │
    ▼                                               │
初始化（每个视频）                                   │
  ├─ 均匀采样 8 帧                                  │
  ├─ DescAgent 生成描述 + 打分                       │
  └─ 建立 Frame Bank                                │
    │                                               │
    ▼                                               │
主循环迭代（最多 10 次）                             │
  ├─ 计算优先级 → 选中最优视频                        │
  ├─ ToolAgent 决策采样位置 + 策略                   │
  ├─ 采样新帧                                       │
  ├─ DescAgent 描述新帧 + 判断终止                    │
  └─ 更新 Frame Bank / Description / 分数            │
    │                                               │
    ▼                                               │
最终回答生成                                         │
  ├─ 筛选关键帧（分数 ≥ 0.5）                        │
  ├─ 添加水印（视频标签 + 时间戳）                    │
  └─ Qwen-VL 生成答案 ───────────────────────────────┘
    │
    ▼
返回带时间戳标记的答案
```
